# SvelteBoy - WASM API & Data Flow

## How Frontend Imports WASM

```typescript
// src/emulator.ts
import {
  initEmulator, runOneFrame, runEmulator,
  getGameFramePtr, setJoypad, getDebugInfo,
  // ... 30+ functions
  memory as backendMemory
} from "../build/backend";
```

`build/backend.js` is auto-generated by AssemblyScript's ESM binding generator:
- Loads `backend.wasm` binary
- Wraps WASM functions with type lifting/lowering
- Handles ArrayBuffer ↔ WASM memory transfers
- Exports `memory` (WebAssembly.Memory instance)

## Data Flow

### Input → WASM

```
Keyboard event (keydown/keyup)
  → inputs.ts: gameInputKeydownHandler()
  → updateInput() → KeyPressMap store update
  → emulator.ts: preRun() → getInputForEmu()  (bit-OR all pressed keys)
  → setJoypad(keys: u8) [WASM call]
  → assembly/io/joypad.ts: SetKeys()
  → Internal joypad state + interrupt request
```

### Frame Execution Cycle

```
emulator.ts: run() [requestAnimationFrame loop]
  → preRun()
      ├─ setJoypad(inputBits)
      └─ initEmulator() [if first frame]
  → runEmulator(deltaTime * EmulatorSpeed)  [WASM]
      → assembly/emulator.ts: Emulator.Run()
          ├─ CPU.Tick()       → execute Z80 instruction
          ├─ Timer.Tick()     → DIV, TIMA
          ├─ PPU.Tick()       → render scanline pixels
          ├─ DMA.Tick()       → sprite DMA if active
          └─ Interrupt.Request() → fire pending interrupts
      → returns EmulatorStopReason
  → postRun()
      ├─ getDebugInfo()    → CPU/PPU/Timer snapshot → debugStores
      ├─ fetchLogs()       → spliceLogs() → debug log messages
      └─ Audio postRunAudio()
```

### WASM → Screen

```
PPU renders 160×144×4 bytes (RGBA) into WASM linear memory
  → getGameFramePtr() → u32 address
  → new Uint8ClampedArray(backendMemory.buffer, ptr, 160*144*4)
  → PlayCanvas.svelte: canvas.putImageData(new ImageData(frame, 160, 144))
```

### WASM → Audio

```
APU synthesizes PCM for 4 channels into double-buffered output
  → getAudioBuffersToReadCount()
  → loop per buffer:
      getAudioBufferToReadPointer(0)  → left channel ptr
      getAudioBufferToReadPointer(1)  → right channel ptr
      new Float32Array(backendMemory.buffer, ptr, sampleCount)
      AudioContext.createBufferSource() → Web Audio API → speaker
  → markAudioBuffersRead(count)
```

## Type Conversions (JS ↔ WASM)

```typescript
// JS ArrayBuffer → WASM (auto-wrapped by bindings)
const buffer = new Uint8Array(romData);
loadCartridgeRom(buffer);

// WASM pointer → JS typed array (zero-copy view)
const ptr = getGameFramePtr();
const frame = new Uint8ClampedArray(backendMemory.buffer, ptr, 160*144*4);

// WASM → JS managed object (copy via bindings)
const info = getDebugInfo();  // returns plain JS object
```

Zero-copy pointer approach (used for frame buffer and audio) is ~10x faster than managed bindings.
